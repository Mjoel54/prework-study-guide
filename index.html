HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The head element contains a <meta> element that sets the default rendering of the HTML in the browser to the device width. This was new to me.
  • Declare a HTML document by using <!DOCTYPE html>. This tells the browser to expect an HTML document.
  • In creating these HTML notes I also learnt about HTML entities so that I can display < and > in the browser inside <code> elements

Git

  1. git status: checks what branch we are currently on
  2. git checkout -b branch-name: creates a new branch and switches to it
  3. git add -A: adds the code to the current working branch
  4. git commit -m "clear message about the update": creates a commit that captures a snapshot of the currently staged changes
  5. git pull origin main: Double check if the local branch is in sync with the main branch
  6. git push origin feature/starter-code: push changes to the remote branch
  7. Go to Github to review changes and create a pull request to merge the branch with the main branch.
  8. Github is tricky, be patient!
