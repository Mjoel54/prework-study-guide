HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The head element contains a <meta> element that sets the default rendering of the HTML in the browser to the device width. This was new to me.
  • Declare a HTML document by using <!DOCTYPE html>. This tells the browser to expect an HTML document.
  • In creating these HTML notes I also learnt about HTML entities so that I can display < and > in the browser inside <code> elements

CSS

  • Link the style sheet to the HTML document by adding <link rel="stylesheet" href="./assets/style.css">
  • Do Not Repeat code
  • Multiple selectors: Separate selectors with commas and new lines to apply the same styling
  • Descendant selectors: Select nested elements using the descendant combinator. Separate selectors with a space and read from right to left, for example all list items within a containing element with a class of "main-list" would be written as, .main-list li

Git

  • Check what branch you are currently on by running git status
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: This command stages all changes in the working directory for the next commit.
  • git commit -m "clear message about the update": creates a commit that captures a snapshot of the currently staged changes
  • git pull origin main: Double check if the local branch is in sync with the main branch
  • git push origin feature/starter-code: push changes to the remote branch
  • Go to Github to review changes and create a pull request to merge the branch with the main branch.
  • Github is tricky, be patient!
  • Investigate Github Copilot

JavaScript

  • A variable is a container that allows us to store data in our code.
  • Create a function by using the function keyword and give the function a unique name.
  • Control flow is the order in which a computer executes code in a script.
  • Print to the console using console.log();
  • Use if statements to introduce conditional logic into application.
